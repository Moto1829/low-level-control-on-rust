# 第3章: OSシステムコール

## 概要

本章では、RustからOSのシステムコールを直接呼び出す方法を学びます。
通常のRustプログラムは標準ライブラリを通じてOSと対話しますが、
低レベル制御が必要な場面では、システムコールを直接呼び出すことで、
標準ライブラリの抽象化を飛び越えたパフォーマンスや細かい制御が可能になります。

```
┌─────────────────────────────────────────┐
│         Rustアプリケーション             │
├─────────────────────────────────────────┤
│     std::fs / std::net / std::io        │  ← 通常はここを使う
├─────────────────────────────────────────┤
│          libcクレート                    │  ← 本章のアプローチ①
├─────────────────────────────────────────┤
│     インラインASM / syscallクレート      │  ← 本章のアプローチ②
├─────────────────────────────────────────┤
│         Linux / macOS カーネル           │
└─────────────────────────────────────────┘
```

## 学習目標

本章を終えると、以下のことができるようになります。

- `libc`クレートを使ってC標準ライブラリの関数を安全にラップする
- インラインアセンブリまたは`syscall`クレートで直接システムコールを発行する
- `open`・`read`・`write`・`close`・`mmap`などのファイルI/O操作を低レベルで実装する
- `fork`・`exec`・`waitpid`を使ってプロセスを生成・管理する
- シグナルハンドラを登録してシグナルに対応する
- ソケットAPIを直接呼び出してネットワーク通信を実装する

## Linux / macOS での違いについて

本章のコードの多くはLinuxを前提としています。macOSでも動作するコードには
その旨を明記しますが、以下の点に注意してください。

| 機能 | Linux | macOS |
|------|-------|-------|
| `io_uring` | カーネル5.1以降でサポート | **非対応** |
| `sendfile` | 対応 | シグネチャが異なる |
| `epoll` | 対応 | **非対応**（kqueueを使う） |
| `O_DIRECT` | 対応 | 限定的サポート |
| syscall番号 | amd64: `/usr/include/asm/unistd_64.h` | `/usr/include/sys/syscall.h` |
| `mmap` | 対応 | 対応 |
| `fork`/`exec` | 対応 | 対応（Appleシリコンに注意） |

> **注意**: 本章のサンプルコードを実行する際は、各セクションの冒頭に
> 動作プラットフォームを明記しています。Linuxを想定したコードをmacOSで
> 実行しようとするとコンパイルエラーまたは実行時エラーになります。

## 環境構築

```toml
# Cargo.toml
[dependencies]
libc = "0.2"

# Linux限定の機能を使う場合
[target.'cfg(target_os = "linux")'.dependencies]
io-uring = "0.6"
```

```bash
# Linuxでのコンパイル・実行
cargo build --target x86_64-unknown-linux-gnu

# macOSでのコンパイル・実行
cargo build --target aarch64-apple-darwin
```
