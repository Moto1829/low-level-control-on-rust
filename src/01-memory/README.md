# 第1章: メモリ管理

## 章の概要

Rustの最大の特徴のひとつは、ガベージコレクタを持たずに**メモリ安全性**を保証することです。
この章では、Rustプログラムがどのようにメモリを扱うかを低レベルの視点から学びます。

「なぜRustのコードはこう書かなければならないのか」という疑問に答えるためには、
コンパイラの背後で何が起きているかを理解することが不可欠です。
スタックとヒープの違いから始めて、構造体のメモリレイアウト、
カスタムアロケータ、そして生ポインタの安全な扱い方まで順を追って解説します。

---

## 学習目標

この章を読み終えると、以下のことができるようになります。

- スタックとヒープのどちらにデータが置かれるかを判断できる
- `std::mem::size_of` / `align_of` を使って型のサイズとアライメントを確認できる
- 構造体のパディングを理解し、メモリ効率の良いフィールド順を設計できる
- `#[repr(C)]` / `#[repr(packed)]` の使い所と注意点を説明できる
- `GlobalAlloc` トレイトを実装してカスタムアロケータを作れる
- `Box::into_raw` / `Box::from_raw` を安全に使い、メモリリークや二重解放を回避できる
- `Vec::into_raw_parts` / `Vec::from_raw_parts` でベクタのバッファを安全に受け渡せる

---

## 前提知識

| 項目 | 目安 |
|------|------|
| Rustの基本文法 | 変数・関数・構造体・トレイトを理解している |
| 所有権と借用 | `move` / `&` / `&mut` の基本を知っている |
| C言語の基礎（任意） | ポインタの概念があると理解が深まる |

> **注意**: `unsafe` コードが登場しますが、各節で「何が危険で、なぜそのコードが正しいか」を
> 丁寧に説明します。`unsafe` は使い方さえ正しければ強力なツールです。

## 各節の内容

### [01. スタックとヒープ](./01-stack-heap.md)

プログラムのメモリ空間の基礎。変数がどこに置かれるかを `size_of` / `align_of` で確認し、
スタックオーバーフローの原因と対策を学びます。

### [02. メモリレイアウトとパディング](./02-memory-layout.md)

構造体のフィールド順がサイズに影響する仕組みを解説。`#[repr(C)]` / `#[repr(packed)]`、
`memoffset` クレートによるオフセット確認、最適なフィールド並び替えを扱います。

### [03. アロケータ](./03-allocator.md)

`GlobalAlloc` トレイトの内部構造と、jemalloc / mimalloc への切り替え方を学びます。
バンプアロケータを自作して「アロケータとは何か」を体験します。

### [04. Box と生ポインタ](./04-box-raw-parts.md)

`Box::into_raw` / `Box::from_raw` と `Vec::into_raw_parts` / `Vec::from_raw_parts` を
使い、FFI や低レベルデータ構造を実装する際の正しいパターンを学びます。

```
プログラムのメモリ空間
┌─────────────────────────────┐  高アドレス
│        スタック              │  ← 01-stack-heap.md

│─────────────────────────────│
│        ヒープ                │  ← 03-allocator.md

│─────────────────────────────│  低アドレス
│        テキストセグメント     │
│   (機械語命令)               │
└─────────────────────────────┘

型のメモリレイアウト            ← 02-memory-layout.md
┌─────┬─────┬─────┬─────┐
│ u8  │ pad │ u16 │ u32 │  パディングあり
└─────┴─────┴─────┴─────┘

Box / Vec の内部              ← 04-box-raw-parts.md
  Box<T>  →  *mut T  (ヒープ上の T を指す生ポインタ)
  Vec<T>  →  (*mut T, len, capacity)
```

## この章で登場する主要な API

| API | 説明 | 節 |
|-----|------|----|
| `std::mem::size_of::<T>()` | 型 T のバイトサイズ | 01 |
| `std::mem::align_of::<T>()` | 型 T のアライメント | 01 |
| `#[repr(C)]` | C 互換レイアウト | 02 |
| `#[repr(packed)]` | パディングなし | 02 |
| `memoffset::offset_of!` | フィールドのバイトオフセット | 02 |
| `std::alloc::GlobalAlloc` | グローバルアロケータトレイト | 03 |
| `#[global_allocator]` | アロケータの登録 | 03 |
| `Box::into_raw` | Box を生ポインタへ変換 | 04 |
| `Box::from_raw` | 生ポインタから Box を再構成 | 04 |
| `Vec::into_raw_parts` | Vec のバッファを取り出す | 04 |
| `Vec::from_raw_parts` | バッファから Vec を再構成 | 04 |
